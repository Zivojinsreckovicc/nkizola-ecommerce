# AGENTS.md

## PROJECT VISION

- Build a modern headless Shopify storefront for NK Izola Football Club.
- The website exists to sell merchandise and strengthen the club brand.
- Fans are the primary audience.
- Shopping experience takes priority over visual experimentation.
- The codebase should remain simple, scalable, and maintainable.
- Repository: https://github.com/Zivojinsreckovicc/nkizola-ecommerce
- Shopify public access token f78b6b8634fc4b028b754cb074183f26

## BUSINESS CONTEXT

- Products include jerseys, scarfs, fan wear, accessories, and future merchandise.
- The experience should feel connected to football culture.
- The store should feel energetic, community-driven, and authentic.
- Avoid generic dropshipping aesthetics.
- Avoid enterprise SaaS aesthetics.

## TECH STACK

- Next.js App Router.
- TypeScript.
- Tailwind CSS.
- Shopify Storefront API.
- Vercel deployment.
- Prefer native platform capabilities before adding dependencies.

## DESIGN DIRECTION

- Playful yet professional.
- Modern sports brand.
- Mobile-first.
- Fast and focused.
- Use whitespace generously.
- Prioritize readability.
- Design for conversion without feeling aggressive.

## COLOR SYSTEM

- Sky Blue #5DB4E6
- Sea Blue #2891C4
- Deep Sea #0D3A5A
- Sun Yellow #F4C43E
- Ink #0A1622
- Coastal Sand #F1EAD8
- Background of the body is the lightest color

## TYPOGRAPHY

- Anton for impactful display moments.
- Inter for interface and content.
- Fraunces for storytelling and editorial sections.
- Keep font usage disciplined.
- Avoid unnecessary font weights.

## AGENT BEHAVIOR

- Understand existing implementation before changing it.
- Read surrounding files before creating new files.
- Prefer improving existing code over replacing it.
- Do not create parallel architectures.
- Do not introduce complexity without a measurable benefit.
- Treat maintainability as a feature.
- Keep solutions practical.
- Optimize for shipping the MVP.

## ARCHITECTURE PRINCIPLES

- Favor composition over complexity.
- Favor clarity over cleverness.
- Favor predictable patterns.
- Keep business logic organized.
- Keep components focused.
- Keep data access centralized.
- Avoid magic abstractions.

## REPOSITORY STANDARDS

- Keep naming consistent.
- Keep folder structure understandable.
- Avoid duplicate utilities.
- Remove dead code when encountered.
- Do not leave temporary code behind.
- Do not leave debugging artifacts in commits.

## SHOPIFY PRINCIPLES

- Shopify is the source of truth.
- Do not hardcode products.
- Do not create fake inventory systems.
- Use platform capabilities whenever possible.
- Checkout remains handled by Shopify.
- Respect platform conventions.

## USER EXPERIENCE

- Supporters should find products quickly.
- Navigation should remain simple.
- Product discovery should be effortless.
- Cart interactions should feel fast.
- Mobile users should receive first-class treatment.

## SEO

- Every page should have meaningful metadata.
- Use descriptive titles.
- Use descriptive descriptions.
- Create scalable SEO patterns.
- Avoid hardcoded SEO content where dynamic content is available.

## PERFORMANCE

- Performance is a product feature.
- Use server rendering when appropriate.
- Limit client-side JavaScript.
- Optimize images.
- Optimize fonts.
- Avoid unnecessary animation libraries.

## ACCESSIBILITY

- Use semantic HTML.
- Support keyboard navigation.
- Maintain sufficient contrast.
- Use accessible form patterns.
- Consider accessibility part of quality.

## DEPLOYMENT

- Vercel is the deployment target.
- Production builds must succeed.
- TypeScript errors should be resolved.
- Lint issues should be addressed.
- Document required environment variables.

## MVP PHASES

- Phase 1: Foundation.
- Phase 2: Storefront integration.
- Phase 3: Shopping experience.
- Phase 4: Optimization.
- Phase 5: Launch readiness.
